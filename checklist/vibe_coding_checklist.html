1The Six Principles of Responsible AI-Assisted Coding
1
Treat AI as an intelligence — give it clear context, domain knowledge, and specific instructions
2
Distrust — assume output is plausible but wrong until independently verified
3
Verify — write unit tests, check edge cases, validate against known-answer cases
4
Prioritize reporting & interpretation — understand every line; if you can't explain it, you don't understand your analysis
5
Engineer adversity — deliberately try to break AI-generated code with bad inputs and edge cases
6
Maximise transparency — document AI usage, log prompts, version code, capture metadata
2Before You Submit — Verification Checklist
I can explain every line of AI-generated code in my own words
I have documented which parts of my code were AI-generated
I have tested with at least one known-answer case
I have captured reproducibility metadata (package versions, dataset versions, dates, parameters)
I have checked units, projections, and scale parameters
I have reviewed the code with a human colleague
I have verified band/variable names against official documentation
I have checked that results are scientifically plausible (order of magnitude, sign, range)
I have tested edge cases (empty regions, zero values, missing data, NaN)
I have saved my AI conversation/prompts for reference
